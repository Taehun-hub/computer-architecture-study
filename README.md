# computer-architecture-study
Study notes and implementations for Computer Architecture

컴퓨터 구조 + 운영체제(1주차)

1. 컴퓨터 구조를 알아야 하는 이유

단순히 코드만 작성하는 게 아니라 컴퓨터를 분석 대상으로 보게 해줌
그래서 문제가 생겼을 때 근본적으로 해결할 수 있는 능력을 키울 수 있음

또한 서버 환경이나 클라우드 사용할 때 성능, 비용, 용량을 고려해서 더 효율적인 선택을 할 수 있게 해줌

2. 컴퓨터 구조의 큰 그림

컴퓨터가 어떤 정보를 다루고 어떤 부품으로 구성되는지 이해하는 게 기본임

2.1 컴퓨터가 이해하는 정보

데이터는 숫자, 문자, 이미지 같은 정적인 정보임
명령어는 데이터를 움직이고 실제로 컴퓨터를 동작시키는 역할을 함

2.2 컴퓨터의 4가지 핵심 부품

CPU는 메모리에 있는 명령어를 읽고 해석하고 실행함
내부에는 ALU, 레지스터, 제어장치가 있음

메모리(RAM)는 현재 실행 중인 프로그램의 데이터와 명령어를 저장함
주소를 통해 위치를 구분함

보조기억장치는 전원이 꺼져도 데이터를 저장함
대표적으로 HDD, SSD가 있음

입출력장치는 외부와 데이터를 주고받는 장치임
마우스, 키보드, 모니터 등이 해당됨

시스템 버스는 컴퓨터 내부 부품들이 데이터를 주고받는 통로임
주소 버스, 데이터 버스, 제어 버스로 구성됨

3. 데이터의 표현 방식
3.1 숫자 표현

2진법은 0과 1로 숫자를 표현하는 방식임
값이 2를 넘으면 자리 올림이 발생함

2의 보수는 음수를 표현하는 방법임
비트를 반전시키고 1을 더하는 방식임

16진법은 0~15까지 표현하는 방식임
2진수와 변환이 쉬워서 자주 사용됨


컴퓨터 구조 + 운영체제(2주차)

3.2 문자 표현

ASCII는 7비트로 문자를 표현하는 방식임
주로 영어 중심임

EUC-KR은 한글을 표현하기 위한 인코딩 방식임

유니코드는 전 세계 문자를 표현하기 위한 표준임
UTF-8은 이를 가변 길이로 저장하는 방식임

4. 명령어와 컴파일 과정
4.1 고급 언어와 저급 언어

고급 언어는 사람이 읽기 쉽게 만든 언어임
C, Python 등이 있음

저급 언어는 컴퓨터가 직접 이해하는 언어임
기계어와 어셈블리어가 있음

4.2 실행 방식

컴파일 언어는 전체 코드를 한 번에 번역해서 실행함
C, C++ 등이 있음

인터프리터 언어는 코드를 한 줄씩 해석하면서 실행함
Python 등이 있음

4.3 C 언어 컴파일 과정

전처리는 매크로 처리와 헤더 파일을 포함하는 단계임

컴파일 단계에서는 어셈블리어로 변환함

어셈블 단계에서는 기계어로 변환함

링킹 단계에서는 여러 파일을 합쳐 실행 파일을 만듦

5. 명령어 구조와 주소 지정 방식
5.1 명령어 구조

명령어는 연산 코드와 오퍼랜드로 구성됨

연산 코드는 어떤 작업을 할지 결정함

오퍼랜드는 데이터 또는 데이터의 위치를 의미함

5.2 주소 지정 방식

제한된 크기의 명령어로 더 많은 데이터를 다루기 위해 사용함

즉시 주소 지정은 데이터를 직접 명령어에 넣는 방식임

직접 주소 지정은 데이터가 있는 주소를 바로 사용함

간접 주소 지정은 주소의 주소를 사용해서 범위를 확장함

레지스터 주소 지정은 레지스터에 있는 데이터를 사용함
메모리보다 빠른 특징이 있음
